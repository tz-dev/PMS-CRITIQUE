A PMS-conform application layer for critique: thresholds, reach, scale, drift, boundedness, and non-coercive correction

1. Purpose and scope of this specification

This document specifies the PMS-CRITIQUE layer in a concise, technical form. It is based on the YAML file PMS-CRITIQUE.yaml (with schema_version = "PMS-CRITIQUE_1.1") and makes its structure, concepts, constraints, and guardrails transparent for human readers and software systems.

PMS-CRITIQUE is an application layer (overlay) of the Praxeological Meta-Structure (PMS; Δ–Ψ). It does not redefine PMS operators, dependency relations, or derived structures. Instead, it provides a critique grammar: strict minimal conditions for critique, thresholds for productive and stable correction, scale effects, predictable drift forms under load (Ω/Θ/Λ/Α), critique-facing reuse of the base PMS derived axes (A/C/R/E/D), and an optional MIP docking interface under strict separation, while enforcing a non-moral, non-clinical application firewall: Χ (Distance) + reversibility + D (dignity-in-practice).

The specification is strong but bounded. It does not claim that PMS-CRITIQUE is final, exhaustive, uniquely sufficient, or superior to all rival critique grammars. It records the current overlay representation for this repository version and remains open to revision, rival framing, and genuine non-capture.

The specification covers, in particular:

  • the schema_meta block (model identity, status, maturity, repositories, authors, inheritance reference, and epistemic boundedness);
  • the PMS reference block (pms_reference) as an explicit non-redefining dependency statement, including methodological restraint;
  • the core critique definitions (CRITIQUE_MIN, CRITIQUE_PROD, CRITIQUE_STABLE) as strict, operator-readable thresholds;
  • the critique-facing projection block (critique_projections) as overlay-level reuse of the base PMS derived axes A/C/R/E/D;
  • the axes of analysis: vertical reach (V0–V4), horizontal scale (paper tags a–d plus optional deployment tags), and the publicness overlay P;
  • a drift typology treated as an open structural index (D1–D6 and compact catalogue), plus typical drift chains;
  • modulators (operator weightings, not person-typing) that shift reachability and stabilization;
  • an optional MIP docking interface with strict non-substitution and no back-projection;
  • limits and boundedness, including structural limits, genuine non-capture, and rival openness;
  • guardrails (validity gate + misuse firewall) and an optional example packet schema for uniform case mappings.
Core idea

PMS-CRITIQUE turns “critique” into an operator-readable configuration with explicit thresholds, scale conditions, and drift sensitivities. It does not judge who is right, does not diagnose persons, and does not prescribe discourse ethics. It models structural viability, structural drift, and bounded structural legibility of critique under asymmetry and temporality.

2.2 Conceptual separation

PMS (base)

PMS defines the operator system (Δ–Ψ), its dependency graph, layers, and derived structures. PMS is the canonical source of operator definitions and derived-axis formulas.

  • Operators and dependencies: Δ–Ψ
  • Layering L1–L4
  • Derived axes (A, C, R, E, D) and other derived PMS structures
PMS-CRITIQUE (overlay)

PMS-CRITIQUE defines critique as a reachable chain, a scale-sensitive formation, and a drift-sensitive configuration under Ω/Θ/Λ/Α. It adds no new operators; it adds overlay-level thresholds, axes, drift types, boundedness clauses, and an optional case schema.

  • Strict minimal critique threshold: Δ + □ + Χ
  • Reach ladder V0–V4
  • Scale ladder a–d + overlay P
  • Drift typology D1–D6 as open structural index
  • Optional MIP docking under strict separation
Non-redefinition rule

PMS-CRITIQUE may provide application-facing glosses (for example, “Χ as stop-capability” or “E as correction reach descriptor”), but it must not modify PMS operator definitions, dependency rules, or derived-axis formulas. Canonical meaning remains in PMS.yaml.

3. Core rules and guardrails

3.1 Validity gate (application firewall)

PMS-CRITIQUE enforces a strict application gate. This gate constrains use of the model, not critique, refusal, revision, or rejection of the model:

  • Χ (Distance): stop-capability must be practically activatable (not merely asserted).
  • Reversibility: readings remain revisable, scene-bound, and non-totalizing.
  • D (Dignity-in-practice): no shaming, humiliation, actor-ranking, or exposure-as-sanction.

3.2 Scope constraints (non-goals)

PMS-CRITIQUE explicitly disallows:

  • person-labeling, motive attribution, diagnosis, or mental-state inference;
  • verdict outputs (“who is right”) as a model product;
  • coercive binding (Ψ→Other), threats, or exposure-as-punishment;
  • legitimacy tribunal logic or sanctioning logic;
  • using drift labels as sanctions rather than as structural descriptors.

3.3 Application invalidity

The YAML now makes explicit that critique becomes structurally invalid as a PMS-CRITIQUE enactment when degradation, humiliation, or exposure steering replaces correction. This invalidity applies to model use, not to the ontological status of the scene or the persons involved.

Misuse hint

Drift language is descriptive, not punitive. Turning “judgment drift” or “public pillory” into a moral badge, ranking token, or enforcement tool violates the layer’s own validity conditions (Χ + reversibility + D).

4. Critique core: strict thresholds (CRITIQUE_MIN / PROD / STABLE)

PMS-CRITIQUE defines critique via operator thresholds rather than rhetorical style, speaker prestige, or speaker intent. The three core definitions form a ladder from interruption to durable correction:

Definition Required signature Structural meaning
Minimal critique
CRITIQUE_MIN		 ["Δ","□","Χ"] A mismatch becomes legible inside a frame and remains interruptible (stop-capable). This is critique in the strict sense used here.
Productive critique
CRITIQUE_PROD		 ["Δ","□","Χ","Φ","Σ"] Recontextualization relocates the mismatch into an actionable context; integration consolidates a coordinatable corrective step.
Stable critique
CRITIQUE_STABLE		 ["Δ","□","Χ","Φ","Σ","Ψ"] Correction becomes durably owned over time via non-coercive self-binding. Follow-up remains revisable and dignity-constrained.

4.1 Threshold notes

  • CRITIQUE_MIN is a strict structural minimum within PMS-CRITIQUE, not a universal ontology of critique.
  • CRITIQUE_PROD marks structural reach toward correction, not the full ethical or political value of critique as such.
  • CRITIQUE_STABLE marks trajectory binding, not certainty, closure, or guaranteed success.

4.2 Non-equivalences

PMS-CRITIQUE explicitly distinguishes critique from common masquerades:

  • Reaction: Δ + ∇ without Χ is not “weak critique” but a different structure.
  • Judgment: □ + Δ without Φ/Σ does not count as critique in the strict sense used here.
  • Silence/paralysis: mismatch without interruptible framing is not low-grade critique but a distinct stabilization.
Reach ladder is not a virtue ladder

“V4 stable critique” is not moral superiority. It only denotes that a chain structurally reaches binding (Ψ) without coercion. Any attempt to use reach levels as status ranking violates the layer.

5. Operator-to-critique mapping (Δ–Ψ → critique roles)

The block critique_mapping.operator_roles maps PMS operators to critique-relevant functions. These mappings are one-to-many and depend on frame and scale. They are descriptive, scene-bound, and non-psychological.

5.1 Mapping logic

  • Δ makes mismatch nameable (what is “off”).
  • ∇ introduces response pressure (tempo, urgency, defense).
  • □ determines what counts as relevant, interruptible, and correctable.
  • Λ encodes meaningful absence (missing replies, omitted follow-ups) and accumulates load.
  • Α stabilizes scripts (outrage, silence, deflection, mockery) that handle pressure without correction.
  • Ω distributes exposure and cost (who risks retaliation, shame, or residue).
  • Θ hardens consequences and narrows revision windows.
  • Φ can bridge to actionability or become narrative substitute when Σ is absent.
  • Χ is stop-capability: pause, revise, re-enter without fusing into discharge.
  • Σ consolidates a coordinatable corrective step.
  • Ψ binds follow-up over time without coercing others.
Dependency hygiene (Χ as reduced marker)

PMS-CRITIQUE uses Χ as a reduced marker for interruptibility. In PMS terms, Χ presupposes Φ and Θ within a □ background. The layer does not treat Χ as a standalone trait.

6. Critique-facing projections (reuse of PMS derived axes)

The block critique_projections reuses the base PMS derived axes A/C/R/E/D as readability aids for critique scenes. These projections do not add new primitives and do not override PMS-derived formulas.

6.1 Reuse status

The YAML now explicitly states that these items are overlay-level reuse of the derived axes specified in PMS.yaml. This means:

  • the formulas remain the base PMS formulas;
  • the projection block is interpretive, not constitutive;
  • critique-facing reuse does not convert the derived axes into new critique primitives;
  • overlay usage does not override base PMS derivation.

6.2 Projection table

Projection Formula Critique-facing use
A Awareness ["Θ","□","Δ"] Sustained, framed differentiation across time.
C Coherence ["Θ","Λ","□","∇"] Narrative and expectation stability under time, framing, and non-events.
R Responsibility ["Ψ","Φ","Θ","Ω"] Bound follow-up ownership under asymmetry over time.
E Action ["Σ","Θ","∇"] Enactable correction rather than commentary.
D Dignity-in-Practice ["Ψ","Χ","Ω"] Restrained handling of asymmetry under self-binding.

6.3 Special note on E

The current YAML adds a specific docking note for E: within PMS-CRITIQUE, E can function as a critique-internal bridging descriptor for correction reach. It is not thereby elevated into a separate MIP core parameter.

Projection note

These projections increase readability within this framework. They reuse the base PMS derived axes for critique-facing interpretation, but do not establish that critique can be exhaustively represented through them in every case.

7. Axes of analysis: reach (V0–V4), scale (a–d), and overlay P

7.1 Vertical axis: reach ladder (V0–V4)

Reach measures how far critique structurally travels before breaking: from mismatch registration to interruption, correction, and follow-up binding.

Level Name Signature Meaning
V0 Irritation only ["Δ"] Mismatch is registered; no structured critique episode emerges yet.
V1 Reaction / discharge ["Δ","∇"] Directional response without stop-capability; critique in the strict sense used here does not occur.
V2 Minimal critique ["Δ","□","Χ"] Interruptible framed mismatch; episode remains revisable.
V3 Productive critique ["Δ","□","Χ","Φ","Σ"] Actionable reframing + integration consolidate a corrective step.
V4 Stable critique ["Δ","□","Χ","Φ","Σ","Ψ"] Correction is bound over time non-coercively; follow-up remains reversible and dignity-constrained.

7.2 Horizontal axis: scale of critique (paper tags a–d)

Scale specifies where asymmetry (Ω: exposure gradients) and temporality (Θ: accumulation and irreversibility) stabilize across scenes. The operator grammar remains the same, but viability conditions change with scale (Χ-availability, Σ-cost, Ψ-feasibility).

  • a = generic / structural critique (no actor binding)
  • b = single-subject critique (self / support / learning)
  • c = small-group critique (peers / team / family)
  • d = public critique (street / media / online)
Chapter marker (quote-ready)

The operators stay the same, but scale decides the cost: as critique moves into publicness, exposure and irreversibility amplify drift without adding new structure.

7.3 Publicness overlay P

PMS-CRITIQUE defines P as an amplification overlay (not an operator). It amplifies:

  • Ω (exposure gradients),
  • Θ (irreversibility / residue),
  • Α (script stabilization speed),

without generating critique or correction by itself. P increases drift sensitivity when chains break early.

8. Drift typology: predictable mutations under load

Drift labels describe structural mutations of critique chains under load (Ω/Θ/Λ/Α and overlay P). Drift is not accusation and not moral ranking. It is a compact, operator-readable way to record common failure modes.

8.1 Methodological status

The current YAML now states explicitly that the drift typology is an open structural index. This means:

  • it is not offered as an exhaustive classification of all critique failures;
  • it indexes recurrent mutation paths that become especially legible in PMS-CRITIQUE terms;
  • some cases may resist clean classification without this constituting mere deficiency;
  • resistant cases may indicate a limit of the typology rather than a flaw in the scene.

8.2 Primary drift classes (D1–D6)

Class Name Minimal signature Structural description
D1 Reaction masquerading as critique Δ + ∇ / Χ absent Impulse discharge replaces interruption; tempo outruns stop-capability.
D2 Judgment masquerading as critique □ + Δ / Φ and Σ absent Evaluation replaces coordination; labels stabilize clarity at the cost of actionability.
D3 Narrative repair masquerading as critique Φ high / Σ absent Recontextualization becomes endpoint; explanation expands while correction remains unreachable.
D4 Commentary / analysis paralysis Χ present / Σ absent Meta-position stabilizes; interruption suspends correction indefinitely.
D5 Public pillory Ω + Θ + Α under P Exposure replaces coordination; interruption fuses with degradation; residue dominates.
D6 Silence attractor Λ persistent (Λ → Α) under Ω/Θ Repeated non-intervention stabilizes as default; non-critique becomes the script.

8.3 Compact catalogue and drift chains

In addition to D1–D6, the schema contains a compact non-exhaustive catalogue (C1–C10) including reaction drift, judgment drift, moralization drift, narrative reset, commentary drift, silence-as-steering, diffusion drift, public pillory, integration bypass, and commitment evasion.

  • Public chain: Ω salience increases → Χ erodes → reaction stabilizes (Α) → Θ hardens residue.
  • Narrative-to-silence chain: Φ inflation delays Σ → attention shifts → Λ repeats → silence stabilizes (Α).

9. Modulators (operator weightings; not person types)

Modulators (M1–M8) are scene parameters that shift reachability and stabilization without adding new primitives. They are not prescriptions and not psychological typing.

The current YAML also makes explicit that the modulator set is intentionally selective: it identifies recurrent weighting conditions that become especially legible in critique configurations, not a complete inventory of all possible scene-load variables.

Modulator Name What it shifts
M1 Frame clarity How clearly □ defines interruption rights, response criteria, and what correction means.
M2 Non-event density How often Λ occurs and how much coordination shifts into absence.
M3 Asymmetry visibility How salient Ω exposure gradients and retaliation risks are in the scene.
M4 Temporal compression Time pressure vs availability of review windows (Θ).
M5 Recontextualization habituation Whether Φ bridges to correction or becomes a narrative substitute (Σ absent).
M6 Distance availability Whether Χ stop-capability is practically activatable without penalty.
M7 Attractor inertia How quickly Α scripts harden into default handling (outrage, silence, mockery, judgment).
M8 Publicness overlay P amplifies Ω/Θ/Α and increases drift sensitivity without changing operator grammar.

10. Optional MIP docking interface

The current PMS-CRITIQUE schema contains an optional MIP docking interface. This is not a merger of models. It is a controlled handoff from structural explanation to explicit evaluation when legitimacy, attribution, or regime conditions are the stated object.

10.1 Non-implication

Docking does not imply that PMS-CRITIQUE naturally culminates in evaluation. PMS-CRITIQUE remains complete as a structural grammar without downstream docking.

10.2 Separation rule

PMS-CRITIQUE role

Explains how critique emerges, stabilizes, drifts, and binds at the level of operator grammar.

MIP role

Evaluates enactments, attribution, and regime conditions under explicit criteria, documented information conditions, and guardrails.

The separation rule blocks:

  • PMS-CRITIQUE being used to decide legitimacy;
  • MIP being used to explain critique emergence or mutation;
  • back-projection of MIP verdicts into PMS causal language;
  • retrofitting PMS structures into evaluative scores.

10.3 Docking points

  • Projection input: critique-facing projections (A/C/R/E/D) may inform MIP-style case description without becoming trait claims.
  • Asymmetry evaluation: PMS may describe asymmetry structurally; MIP may evaluate enactment under its own criteria.
  • Regime/publicity constraint: publicness and transmission conditions may act as admissibility constraints for evaluation.
  • Shared critique-readiness: docking is valid only when both explanation and evaluation remain documented, reversible, bounded, and non-weaponizable.

11. Limits, non-capture, and rival openness

11.1 Structural limits

PMS-CRITIQUE explicitly does not:

  • establish truth;
  • verify motives, sincerity, or inner states;
  • adjudicate moral rightness;
  • rank persons, roles, or groups;
  • function as a clinical, forensic, HR, compliance, or sanctioning tool;
  • dissolve accountability into “just structure”;
  • promise exhaustive capture of lived experience, normative disagreement, or social reality.

11.2 Genuine non-capture

PMS-CRITIQUE does not assume that every critique-relevant phenomenon can be adequately rendered within its own chain logic. Genuine non-capture remains possible where:

  • the grammar yields a coherent reading but weakens what is structurally decisive in the case;
  • critique functions more as symbolic refusal, public rupture, ethical contestation, or resistance than as correction-oriented coordination;
  • rival framings preserve dimensions that PMS-CRITIQUE backgrounds by design.

The anti-absorption rule blocks three common overextensions:

  • do not prematurely redescribe resistant cases as mere drift;
  • do not treat resistant cases as calibration noise by default;
  • do not treat incomplete capture as proof that the scene is defective.

11.3 Rival openness

Rival critique grammars remain methodologically open. PMS-CRITIQUE may be more structurally legible in some cases without thereby being superior in all respects.

Final boundedness statement

Nothing in this schema requires PMS-CRITIQUE to be treated as final, exhaustive, or uniquely sufficient. Its strength remains most credible where its own limits, rival framings, and possible non-capture cases are kept explicit.

12. Example packets: optional case schema for uniform mappings

PMS-CRITIQUE optionally defines an instance-facing case schema (example_suite.packet_schema) to keep case mappings comparable across analysts and tools. Example packets are illustrative; they do not define semantics.

12.1 Required fields

  • scene_label
  • frame_anchor (□)
  • trigger_mismatch (Δ)
  • operator_signature (reduced; 2–6 operators)
  • guardrail_gate (Χ + reversibility + D reminder)
  • structural_closure (2–5 plain sentences; non-instructional)
  • scale (system + tag; legacy scale_tag supported)

12.2 Optional fields

  • impulse_pressure (∇)
  • non_event_candidates (Λ)
  • drift_class (D*) and up to two drift markers
  • cost_distribution (Ω/Θ)
  • publicness_overlay (P) note
  • reach_level (V0–V4)
  • modulators (M1–M8)
  • projection_notes (A/C/R/E/D)
  • non_capture_flag
Constraint

Packets must remain role/scene/constraint focused. No motives, no traits, no diagnosis, no “should”. Non-capture flags must not be used as dismissal tokens.

13. Implementation notes, integration, and citation

13.1 YAML and integration

The official PMS-CRITIQUE YAML specification is provided as PMS-CRITIQUE.yaml. It is intended to be loaded as an addon layer after the PMS base grammar.

13.1.1 Recommended load order (non-normative)

PMS.yaml → PMS-CRITIQUE.yaml → pms-critique-ext.yaml (optional)

13.1.2 What an agent/tool can do with PMS-CRITIQUE (beyond PMS alone)

  • classify episodes along reach (V0–V4) using strict operator thresholds;
  • map scenes along scale (paper tags a–d) and annotate viability shifts under Ω/Θ;
  • identify drift forms (D1–D6 and compact catalogue) as operator-readable mutations;
  • reuse A/C/R/E/D as critique-facing projections without overriding PMS base formulas;
  • record modulators (M1–M8) as non-typing scene parameters;
  • represent boundedness, non-capture, and rival openness explicitly;
  • produce comparable outputs via optional example packets;
  • optionally dock to MIP under strict separation.

13.1.3 Recommended bootstrap for LLM-based agents (non-normative)

After loading the YAML, an agent SHOULD:

  • activate the guardrails (Χ + reversibility + D) as interaction constraints;
  • require scene-bounded inputs and refuse person-typing outputs;
  • when useful, output reach level, scale tag/system, operator signature, drift class (if any), modulators, and boundedness-related notes grounded in the schema;
  • treat critique-facing projections as reuse of PMS derived axes, not as newly defined primitives;
  • avoid treating structural fit as proof of exhaustive capture;
  • treat MIP docking as optional and non-substitutive.
